# 3961. 设备评分的最大和

## 关键点
1. 每个设备只需要关注单元最小的两个容量就好了，其他容量没有意义，因为设备的评分要不是最小的容量，要不就是把最小的容量抛弃给别的设备，第二小的容量留下来作为评分；
2. 每个设备可以接受无限个其他设备丢来的单元容量。
## 解法
### 1.贪心
根据上面的两个关键点，我们可以看出：
1. 每个设备只用保留最小的评分和第二小的单元容量；
2. 所有设备中最小评分的最小值必须要有设备来作为最终的评分，至于给哪个设备就不一定了。
3. 将所有设备中第二小的容量从大到小排列，用这些容量作为设备的评分，最后一个设备的评分只能用2中选出的最小评分。这样我们就可以求出整体的最大值。
#### 时间复杂度
```math
O(m*n)
```
#### 空间复杂度
```math
O(1)
```
####  代码
```python
class Solution:
    def maxRatings(self, units: List[List[int]]) -> int:
        ret = 0
        min1,min2=100001,100001
        m = len(units)
        n = len(units[0])
        if n == 1:
            for i in range(m):
                ret = ret + units[i][0]
            return ret
        #这里我尝试了迭代器和下标两种方式，发现本身自带的迭代器速度要快一些
        for line in units:
            linemin1,linemin2 = 100001,100001
            for node in line:
                if node < linemin1:
                    linemin2 = linemin1
                    linemin1 = node 
                elif node < linemin2:
                    linemin2 = node
            ret = ret + linemin2
            min1 = min(min1,linemin1)
            min2 = min(min2,linemin2)
        ret = ret - min2 + min1
        return ret
```
